LivrosAPI

Documentação

A documentação vai te ajudar a melhor entender o funcionamento da nossa API e garantir que você possa aproveitar o máximo possivel dela.

GET

localhost:3000/Api/livros

O método GET é o responsável por exibir os livros registrados em nossa API. Nele não é obrigatório ser feito uma passagem de parâmetros para que ele possa ser executado com sucesso, mas ele irá retornar todos os livros que estiverem registrados. Para buscar um livro especifico pode ser passado o seu id como parâmetro.

Usando parâmetros

Para passar o parâmetro basta adicionar a / e o número do id que deseja buscar no caso do exemplo vamos usar o 2. localhost:3000/Api/livros/2.


POST

localhost:3000/Api/livros

O método POST é o responsável fazer a requisição de armazenamento para que os dados dos livros sejam registrados. Nele é preciso ser passado os dados para que possam ser registrados e esses dados devem ser passados em formato JSON como podemos ver no exemplo abaixo.


PUT

localhost:3000/Api/livros

O PUT é o método que usamos para alterar um dado que já foi registrado no caso da nossa API um livro. No método PUT nos devemos primeiramente fazer o GET para sabermos qual livro vamos alterar depois é só pegar os dados e alterá-los. Se você lembrar do nosso primeiro exemplo no método GET usamos o livro de id 2 chamado Técnicas de invasão. No exemplo abaixo nos vamos alterar seu nome.


DELETE

localhost:3000/Api/livros

O método DELETE apaga todos os registros de livros salvos.

Usando parâmetros

Para apagar somente um livro é necessário passar o id do livro como parâmetro. No exemplo vamos usar o id de número 2. localhost:3000/Api/livros/2
